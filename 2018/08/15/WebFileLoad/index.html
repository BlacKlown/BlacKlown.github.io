Html,CSS,JS文件加载顺序

发表于 2018-08-15 | 分类于 WEB前端

今天看了看HTML文件加载的问题，打算记点笔记。
大体流程关于<link>、<script>和<img>三个标签。

主体加载流程

  1. 浏览器由上至下加载html文件。
  2. 加载到<link>标签，浏览器向服务器发送请求获取.css文件，然后html文件和css文件并行加载。
  3. 加载到<script>标签，如果没有引用外部文件就立即执行。
  4. 如果<script>标签引用了外部文件，就下载外部文件，并阻塞其他代码的下载和解析。
  5. 加载到<img>标签，向服务器发送请求下载图片，此时浏览器不会等待图片下载完成，而是继续渲染余下代码。
  6. 服务器返回图片，此时图片需占用一定的空间，这会影响到后面段落的排布，于是浏览器又重新渲染这部分代码。

JS文件会阻塞DOM树解析和渲染，CSS文件不会阻塞DOM树解析，但是CSS文件会阻塞DOM树渲染，并且阻塞后面JS语句的执行。

优化加载、渲染速度，提高用户体验

HTML

  1. 减少无意义的空格符、换行符、制表符。
  2. 把内联、行内的CSS、JS代码改为外联文件。
  3. 尽量少引用不同域名的文件。

CSS

  1. 尽量把样式文件合并，减少请求次数。
  2. 把首屏的样式文件独立出来，可以减少白屏时间。
  3. 对CSS文件进行压缩

JS

  1. 进行文件合并，减少请求次数。
  2. 将<script>标签放在body的最后，可以避免“阻塞效应”。
  3. <script>标签添加属性defer="true"，其作用为：告诉浏览器等到DOM加载完成后，再执行脚本。可使JS文件下载与HTML解析并行执行。
  4. <script>标签添加属性async="true"，其作用为：告诉浏览器该JS文件不会修改DOM和样式，下载完成后立即执行。可使JS文件下载与HTML解析、CSS渲染并行执行。

当defer与async同时设置时，浏览器会忽略defer属性，其行为由async决定。

img

  1. 指定img的大小，当其加载完成后不用再对后方段落重新渲染。
  2. 使用图片精灵技术，减少图片请求。
  3. 用CSS代码实现简单图片效果。
